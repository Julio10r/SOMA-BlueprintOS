# PROJECT PHILOSOPHY

## Objetivo

Este documento define como o projeto deve ser conduzido.

Ele tem prioridade sobre sugestões espontâneas de qualquer IA.

---

# Filosofia

O projeto será desenvolvido de forma incremental.

Cada sprint deve resolver um problema real do produto.

A prioridade é entregar valor continuamente.

Não desenvolver funcionalidades apenas porque podem ser úteis no futuro.

Generalizações só devem acontecer quando houver necessidade comprovada.

---

# Fluxo Oficial

Toda sprint seguirá obrigatoriamente o fluxo abaixo.

1. Propor
2. Sugerir melhorias
3. Aprovar
4. Implementar
5. Validar
6. Finalizar

Nenhuma sprint deve mudar de objetivo durante sua execução.

---

# Escopo

Resolver primeiro o problema atual.

Novas ideias devem ser registradas no backlog.

Nunca aumentar o escopo da sprint atual.

---

# Simplicidade

Sempre escolher a solução mais simples que:

- seja sustentável;
- seja clara;
- seja fácil de manter;
- resolva o problema.

Evitar overengineering.

---

# Documentação

A documentação deve existir porque alguém realmente irá utilizá-la.

Não criar documentação apenas por organização.

Todo documento deve possuir um público definido.

---

# Arquitetura

A arquitetura deve evoluir junto com o produto.

Não criar componentes genéricos sem necessidade.

O BlueprintOS cresce conforme o +Compras evolui.

Nunca o contrário.
